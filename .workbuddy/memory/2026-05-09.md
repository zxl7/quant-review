## 修复 ./qr.sh fetch 全量更新 ✅

### 问题
用户执行 `./qr.sh fetch` 进行全量更新时，脚本在 `fetch_stock_themes` 函数中崩溃，报错：
```
urllib.error.HTTPError: HTTP Error 404: Not Found
```

### 原因
某些股票代码（如 `920200`）没有题材数据，必盈 API 返回 404，导致脚本崩溃。

### 修复
修改 `daily_review/data/biying.py` 中的 `fetch_stock_themes` 函数，添加 404 错误处理：
```python
def fetch_stock_themes(client: HttpClient, *, code6: str) -> list[dict[str, Any]]:
    import urllib.error
    url = f"{client.base_url}/hszg/zg/{code6}/{client.token}"
    try:
        data = client.get_json(url)
    except urllib.error.HTTPError as e:
        if e.code == 404:
            return []
        raise
    return data if isinstance(data, list) else []
```

### 验证结果
✅ `./qr.sh fetch` 现在能正确处理 404 错误并继续运行
✅ 成功生成报告：`html/复盘日记-20260508-tab-v1.html` (448KB)
✅ 遇到 `code6='920200'` 时正确跳过：`[DEBUG] fetch_stock_themes: 404 for code6='920200', skip`

## 三个问题修复 ✅

### 问题1：上证指数百分比计算异常 → 显示 -0.00%
**原因**：`cli.py` 中用日K `(c-pc)/pc*100` 计算涨跌幅，极小负值（如 -0.001%）格式化后显示为 `-0.00%`
**修复**：在 `cli.py` 三处 chg 计算后添加 `if abs(chg) < 0.005: chg = 0.0` 归零逻辑
- `run_fetch_and_rebuild` 中 `_kline_index` 分支
- `run_fetch_and_rebuild` 中 `indices_rt` fallback 分支（新增 `_norm_index_chg` 辅助函数）
- `run_rebuild` / `run_partial` 中的 `_pick_exact` 分支
- `run_intraday_snapshot` 中的同样逻辑

### 问题2：盯盘工作台数据横向平铺 ✅（上次已完成）
- 模板中 `wb-list` + `wb-row` 替换为 `wb-grid` + `wb-card2` 横向卡片布局
- CSS 新增 `.wb-grid`, `.wb-card2`, `.wb-time`, `.wb-label`, `.wb-metrics`, `.wb-m` 样式

### 问题3：fetch 不记录盯盘快照
**原因**：`run_fetch_and_rebuild`（`./qr.sh fetch`）不调用 `watch_runtime.append_intraday_slice`，只有 `run_intraday_snapshot` 才追加快照
**修复**：在 `run_fetch_and_rebuild` 末尾，`run_rebuild` 之后，构建 watch snapshot dict 并调用：
- `append_intraday_slice(root, snapshot)` → 写入 cache/intraday_slices-YYYYMMDD.json
- `publish_runtime_files(root, snapshot, slices)` → 写入 html/latest_intraday.json + html/latest_intraday_slices.json

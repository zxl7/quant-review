## 修复 ./qr.sh fetch 全量更新 ✅

### 问题
用户执行 `./qr.sh fetch` 进行全量更新时，脚本在 `fetch_stock_themes` 函数中崩溃，报错：
```
urllib.error.HTTPError: HTTP Error 404: Not Found
```

### 原因
某些股票代码（如 `920200`）没有题材数据，必盈 API 返回 404，导致脚本崩溃。

### 修复
修改 `daily_review/data/biying.py` 中的 `fetch_stock_themes` 函数，添加 404 错误处理：
```python
def fetch_stock_themes(client: HttpClient, *, code6: str) -> list[dict[str, Any]]:
    import urllib.error
    url = f"{client.base_url}/hszg/zg/{code6}/{client.token}"
    try:
        data = client.get_json(url)
    except urllib.error.HTTPError as e:
        if e.code == 404:
            return []
        raise
    return data if isinstance(data, list) else []
```

### 验证结果
✅ `./qr.sh fetch` 现在能正确处理 404 错误并继续运行
✅ 成功生成报告：`html/复盘日记-20260508-tab-v1.html` (448KB)
✅ 遇到 `code6='920200'` 时正确跳过：`[DEBUG] fetch_stock_themes: 404 for code6='920200', skip`

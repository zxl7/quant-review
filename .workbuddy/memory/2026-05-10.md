## 个股题材匹配：双重比对（板块反查 + bigram fallback）

用户洞察：板块数据（plateRotateTop.leaders / themePanels.examples）已经包含该板块的最强个股，涨停票几乎一定在其中。因此可以从板块→个股反向构建精确索引，避免 bigram 模糊匹配。

实现：
1. **`themeReverseMap`**（code/name → [{themeName, strengthRow}]）：
   - 从 `plateRotateTop` 的 `leaders`（含 code+name）构建
   - 从 `themePanels.ztTop/zbTop/dtTop` 的 `examples`（名称列表）补充
   - 支持 code 精确匹配 + name 匹配双重键
2. **`getThemeEntries(code, name)`**：查询函数，返回按 net 排序的题材列表
3. **个股匹配优先级**：先 `plateThemes = getThemeEntries()` 精确匹配，命中则直接拿到 strengthRows 标准名 + net/risk/zt 数据；未命中 fallback 到 bigram
4. **`themeLbNames`/`themeLeader` 双重索引**：构建时同时用 codeThemes 名和板块反查的标准名做 key
5. **`fuzzyMatch(a, b)` 统一函数**：替代分散的 bigram 匹配代码，用于 hasTier/matchedStrengthName/isMain
6. 标签文字从"题材推测XX"改为直接显示板块名，reasonBits 从"题材推测"改为"板块归属"

核心优势：通过板块反查拿到的 `topThemes.name` 直接就是 strengthRows 的标准名，不再需要名称对齐。
